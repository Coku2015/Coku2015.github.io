这个架构，仅限Veeam

VBR有个很强大的特性，这个特性深受广大Veeam资深铁粉们喜爱，那就是便携式备份存档(Portable backups)。因为这个特性，VBR的备份存档和备份服务器完全没有关系，也正是这个强大的特性，能够让Veeam的解决方案与众不同。今天我想来说说这个特性能干啥，它能为我们设计备份系统，搭建备份架构时带来什么帮助。

简单来说，利用这个特性，当我们通VBR Server A完成了数据的备份后，我们可以将写在备份存储库中的备份数据，直接导入到一台VBR Server B中。这时候，VBR Server B完全不需要配置任何VBR Server A的数据库记录，备份存档可以被直接识别和使用。这就像压缩包和压缩软件的关系，VBR服务器和VBR的备份存档完全没直接的依赖关系，存档文件可以被任何一套VBR服务器读取和使用。
